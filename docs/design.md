# 設計ドキュメント

## 要件定義

### 扱う商品
晴れ
雨
曇り

### なぜこの商品にしたか
小さい頃にテレビで見たお天気自動販売機を再現したかったため。
メルティーキッスは好きなお菓子なので追加した。

### 受け付ける硬貨
1円・5円・10円・50円・100円・500円

### 各種ルール
釣銭が作れない場合：購入不可にする
売り切れの場合：ボタンを無効化して表示を変える
途中キャンセル：可能。投入金額を全額返金する


 商品データ
配列でオブジェクトを管理（id, name, price, stock）

### 釣銭在庫
オブジェクトで硬貨の枚数を管理（例：{ 500:5, 100:10 ... }）



### なぜ非同期にしたか
実際の自販機はお金を入れたり商品を選んだりするときに
少し時間がかかる。そのタイムラグをsleepで再現するために非同期にした。
